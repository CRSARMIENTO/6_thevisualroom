OpenFOAM and IDE's

IDE stands for integrated development environment. Often programmers use these tools to help optimise their workflow and make overall programming experience slightly less consuming. IDE usually has useful properties such as autocompletion, tree structure of the code, indexing of the code that helps navigating the code, higlighting, etc. All of these benefits pose an obvious question - can we use an IDE program to help us write OpenFOAM code? With some caveats, the answer is YES.

There are various different programs that can be used to achieve IDE experience. I personally have tried using Eclipse IDK, QtCreator and eventually settled with Tmux + Vim. Below there will be a quick setup guide for these three environments, but as with almost every solution in programming it is not the only and one might find that some different setup works for them better.

Eclipse

Eclipse out of all of three environments, probably has the most features. Unfortunately, it comes with a slight caveat: I personally have found that it is quite taxing on your computer resources. so, for instance, it can be come very noticeable on a slower laptop. Nonetheless it is possible to fully integrate OpenFOAM code development inside Eclipse and have:

  • Highlighting
  • Function indexing which gives autocompletion as well as ability quickly open the *.H/*.C file where the function is defined (highly useful for newcomers).
  • Compilation window
  • Debugging/Profiling tools
  • Tree structure of the code folder


"OK, OK, Ben you sold it to me. How do I start working with it?"

First things first - download it from: https://www.eclipse.org/downloads/

Then, my recommendation would be to run through OpenFOAM wiki guides on how to integrate FOAM code to eclipse:

  • How to use OpenFOAM with Eclipse
  • How To Use OpenFOAM with Eclipse/Fool the indexer

As well as check out this Youtube tutorial:

  • How to use Eclipse as IDE for extending OpenFOAM® - 1 Basic Setup

These should be sufficient to get everything working. My setup was slightly different so I have written my steps down.

Eclipse setup by Ben

  1. Install current Eclipse version
  2. Upon opening eclipse choose a workspace folder (eg. $HOME/workspace/ ). Below a text box, tick a checkbox to set this workspace as a default.
  3. In Eclipse-Neon a welcome page pops up, close it to get to project view.
    • Right click in 'Project Explorer' space and choose 'Import'.
    • Under C/C++ choose 'Existing Code as Makefile Project' and click 'Next'.
    • In the next window click browse and select OpenFOAM folder. As I wanted to have a whole OpenFOAM source code shown in tree view. To do that I have imported all of OpenFOAM-3.0 directory. This helps to avoid step describing lnInclude specification that is described in OF Wiki guide. The downside of this approach is that indexer takes additional computer resources.
    • Right click on the project in the project explorer section. Select Properties.
    • Under 'C/C++ General'->'Indexer', click 'Enable project specific settings'.
      • In the same window 'Indexer options' section is now editable. Select 'Index unused headers as C++ files' and 'Index all header variants'.

I decided not to bother with Debugger/Profiler/Compiler setup as terminal is still sufficient for this purpose.

If you've done all of these steps then everything should be working! And you can enjoy IDE experience with all of it's benefits.

N.B.: Eclipse supports different plugins so one could add some additional functionality (such as less eye straining color schemes, or additional highlighting options). Everything is done through eclipse marketplace (accessible through Help->Marketplace); setup procedures can be found online.

QtCreator

As mentioned before, from my experience Eclipse tends to be slightly too taxing on my laptop. To avoid that, an alternative can be used - QtCreator. This IDE might not be as feature rich (or at least not that I know of) as Eclipse, but the speed increase was noticeable.

Setup guide:

  1. Download: https://www.qt.io/download-open-source/
  2. Install
    • Change permissions on downloaded file with chmod 755 $FILENAME.run and run the executable
    • In the installation wizard I have selected the latest version of Qt
  3. Upon opening QtCreator click 'New Project'
  4. Under 'Projects' select 'Import Project', then in the following section select 'Import Existing Project' and click 'Next'
  5. In the following window browse OpenFOAM install location and choose project name and click 'Next'
  6. In the following under 'Show files matching:' add '*.C; *.H'
    • In the section below tick all of the file checkboxes
    • Click next
  7. In the following window click 'Finish'
  8. Go to 'Tools->Options->C++->File Naming' and select suffixes as 'H' and 'C'

After a few minutes the IDE should finish indexing the code and you can use the magical F2 key to pull up definitions of functions and data types. Additional bonuses such as highlighting and less eye straining colorschemes can be adjusted in the options menu. Info on how to do that can be found online.

tmux + Vim

"The Gods of Vim and tmux have spoken to me and shown me the way." - Unknown source.

This is the IDE setup that I have settled down with. It uses a combination of what is called a terminal multiplexer and vim.

Terminal multiplexer

What is a terminal multiplexer? It lets you switch easily between several programs in one terminal, detach them (they keep running in the background) and reattach them to a different terminal. In addition to having multiple detachable sessions running at the same time these applications often offer window splitting functionality which greatly helps to organise the workspace and avoid a bunch of floating windows hiding from you.

Couple of examples of such apps are: GNU Screen, Terminator, tmux. Personally I have settled with tmux.

  1. Installing tmux on Ubuntu: https://raw.githubusercontent.com/tmux/tmux/master/README
  2. Installing tmux on Scientific Linux machine (no root access):
    • Ask Rob to install it for you (tmux + all necessary libraries)
    • Alternatively, a slightly more involved option is to use the script that someone has written: https://gist.github.com/ryin/3106801
      • Note that it has been written in 2012, so you might need to change version numbers used in the script to the latest ones available right now.

Now that tmux is installed test it and see if it runs: type 'tmux' in terminal window and click (Ctrl+b, ") this should do a horizontal pane split.

Configuration

  1. As with most of terminal based applications configurations are stored in dotfiles (eg. .bashrc). Luckily there are a lot of examples uploaded online, but to give you a head start I have my tmux configuration files on bitbucket so feel free to steal it (which I did myself as well): http://bit.ly/2m3WBAt
  1. Copy .tmux.conf.symlink to $HOME/.tmux.conf. In order to use the theme file you can copy theme.sh wherever you like and edit tmux conf file at the bottom of it to point to that theme file.
Vim

It turns out that this little text editor can be change to work almost as anything. Using different plugins we can make Vim work and behave almost the same as Eclipse and Qt does, but with almost no strain on your computer.

Everything can be done with help of Vundle - a plugin manager to vim. To install it follow the instructions on: https://github.com/VundleVim/Vundle.vim

Plugins are then specified in .vimrc. In order to achieve IDE like functionality I use the following plugins:

  1. AutoCompletion (https://goo.gl/6WpE23) - YouCompleteMe
  2. OpenFOAM Highlighting (https://bitbucket.org/shor-ty/vimextensionopenfoam)- vimextentionopenfoam
  3. File tree browser (https://goo.gl/KtrJCk) - NerdTree
  4. File searcher (https://goo.gl/zhc5hA) - LustyExplorer
    • As it exists in Vundle repo, just add Plugin 'LustyExplorer' to your .vimrc
  5. Grep function in vim (https://goo.gl/pbb66f) - ack.vim
    • As it exists in Vundle repo, just add Plugin 'ack.vim' to your .vimrc
    • Terminal prerequisite ack-grep : In terminal run sudo apt install ack-grep
    • Terminal prerequisite vim-nox (as far as I understand it's vim ruby support): In terminal run sudo apt install vim-nox

Again, feel free to grab my .vimrc from the bitbucket ( http://bit.ly/2m3WBAt).

Finalising the IDE

  1. To get the plugins added to your vim. Open vim and run ':PluginInstall'.
  2. To get the autocompletion follow the guidelines in https://goo.gl/6WpE23
    • For Ubuntu users:
      1. sudo apt-get install build-essential cmake
      2. sudo apt-get install python-dev python3-dev
      3. cd ~/.vim/bundle/YouCompleteMe
      4. ./install.py --clang-completer
    • For scientific linux users:
      1. Will need to go and speak with Rob. I was not able to do it myself.

And that is it. tmux+Vim now should work as good as does Eclipse and Qt.
